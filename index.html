HTML

  • head contains information that helps render the page correctly
  • elements included within the head are not visible when a user looks at a webpage
  • style element defines CSS styles
  • base element defines URL
  • link element connects external resource to html doc
  • meta element defines metadata (character set, desc, keywords, viewport, etc.)
  • DOCTYPE tells browser what programming language to expect (html)
  • charset sets char encoding
  • http-equiv sets http header
  • viewport sets default view for webpage
  • body contains content shown to user
  • elements user interacts with will be placed inside this element
  • semantic HTML element (name of element and role)
  • h elements represent level of heading
  • p element represents block of text
  • ul represents unordered list, ol represents ordered list
  • img contains information about images that are displayed on the webpage
  • src defines location of img
  • alt contains text string that describes img
  • br makes a line of space
  • a (anchor) creates link
  • href points to url
  • footer contains author, copyright, contact, sitemap, and navigation
  • section defines section in doc that contains content
  • id identifies elements and selects them

CSS

  • there are three ways to style a webpage with css: inline, internal stylesheet, and external stylesheet
  • link connects the external stylesheet to the HTML code
  • structure of a CSS rule: selector {property: value; property: value; etc}
  • margin indicates how much space we want around the outside of an element
  • padding adds space around the inside of an element
  • * is a selector that selects the whole page
  • img selector selects any image in the markup
  • . is a class selector

Git

  • git status tells us what branch we are currently in
  • git checkout moves us to an existing branch
  • git checkout -b makes a new branch
  • git add -A adds modifications in the current working branch to the staging area
  • git commit -m adds a commit message
  • git pull pulls a branch from the repository
  • git push pushes changes to remote branch

Javascript

  • a variable is a named container that allows us to store data in our code
  • var declares a variable
  • console.log prints in the console on the web browser
  • script links the javascript file to the HTML file
  • control flow is the order in which a computer executes the script
  • if statements are used to introduce decision-making into code
  • === check to see if two values are equal, and returns a Boolean result
  • an array is a single variable used to hold a group of data
  • an index is a unique number used to identify data in an array
  • a for loop allows a single code block to be executed over and over
  • the first statement in a for loop determines the starting point of the loop
  • the second statement in a for loop is the condition
  • the final statement allows the array to iterate over each item
  • a function is a set of instructions that tells the computer how to perform a certain task
  • functions are defined first, then called
  • Math functions can be used to generate random numbers
